1. DESIGN SYSTEM & BRANDING
================================================================================

FONTS:
- Headings: Playfair Display (serif) — Google Fonts
- Body: Inter (sans-serif) — Google Fonts

COLOR PALETTE:
- Parchment (background): #F9F7F2
- Ink (text): #1A1A1A
- Crimson (accent): #BC1E22
- White: #FFFFFF
- Muted/border tones: warm greys

DESIGN TOKENS (index.css):
- --font-heading: 'Playfair Display', serif
- --font-body: 'Inter', sans-serif
- --primary: crimson (HSL 358 74% 43%)
- --background: parchment (HSL 36 33% 97%)
- --foreground: near-black (HSL 0 0% 10%)
- --border: HSL 30 15% 88%

VISUAL STYLE:
- Editorial magazine aesthetic inspired by Maju.sg
- Minimal, clean, high whitespace
- Sharp edges (rounded-none buttons, square cards)
- Crimson accent used sparingly as a punctuation mark
- Thin 1px crimson/muted horizontal rule dividers
- All-caps, widely-tracked labels using font-body
- Playfair Display headings, often with italic crimson spans
- Framer Motion scroll-triggered and entrance animations on all sections
- Responsive: mobile-first, breakpoints at md and lg

================================================================================
3. DATABASE ENTITY: Article
================================================================================

Entity name: Article
Fields:
- title (string, required) — Article headline
- slug (string) — URL-friendly slug, auto-generated from title
- excerpt (string) — Short summary
- content (string) — Full HTML content (rich text / WYSIWYG)
- category (enum: legal_insights | opinion | case_study | news | explainer)
- cover_image_url (string) — Cover image URL (uploaded file)
- video_url (string) — Optional embed URL (YouTube, Vimeo)
- instagram_link (string) — Link to related Instagram post
- author (string) — Author name
- is_featured (boolean, default false) — Shows in featured slot on homepage
- is_published (boolean, default false) — Controls public visibility
- published_date (date string) — Display date

--- 4A. HOME PAGE (pages/Home.jsx) ---

Data: Fetch all published articles (is_published: true), sorted by -published_date
Featured article = first article where is_featured=true, fallback to first article
Rest = remaining published articles (up to 6)

SECTIONS:

1. HERO (full-viewport, 85vh min):
   - Full-bleed background image (editorial atmospheric photo)
   - Dark gradient overlay: from-ink/80 via-ink/50 to-transparent (left to right)
   - Content left-aligned, max-w-xl
   - Thin crimson 1px line divider before heading
   - Heading: "Illuminating the Law, One Story at a Time." (white, Playfair Display, 4xl–6xl)
   - Subheading body text: mission statement in white/70
   - CTA button: "Explore Articles →" — crimson background, white text, uppercase tracked, links to /articles
   - Entrance animation: fade + slide up (framer-motion)

2. FEATURED ARTICLE (if exists):
   - Section title: "Featured" with crimson/20 horizontal rule extending right
   - Uses <ArticleCard featured /> in a full-width layout (16/9 image ratio)

3. LATEST ARTICLES (if any non-featured articles):
   - Section title: "Latest" + "View All" link to /articles
   - 3-column grid (1 col mobile, 2 col md, 3 col lg)
   - Uses <ArticleCard /> for each

4. INSTAGRAM FEED (if any articles have instagram_link):
   - Section: "From Our Instagram"
   - Mosaic grid of article cover images (2–4 col responsive)
   - Each tile links to article.instagram_link (new tab)
   - Hover: image dims + overlay with title + ExternalLink icon
   - Footer link: "Follow us on Instagram" → https://www.instagram.com/clairdeloi

5. EMPTY STATE (if no published articles):
   - "Coming Soon" heading + message

ARTICLE CARD COMPONENT (components/articles/ArticleCard.jsx):
- Used on Home, Articles pages
- Link to /article/{article.id}
- Image: aspect-4/3 (standard) or 16/9 (featured), cover-fit, scale-down on group-hover
- Category badge: crimson background, white text, uppercase tracked, absolute top-left over image

- "Read More" label + animated crimson underline growing on hover

  - Each card: icon + large number + label

  - Clicking an article → /admin/articles?edit={id}

ADMIN ARTICLES (pages/admin/AdminArticles.jsx):
- List view: ArticleList component
  - Thumbnail, title, featured star, category, date, IG indicator
  - Published (green) / Draft (muted) badge
  - Edit (pencil icon) and Delete (trash icon with AlertDialog confirmation) actions

- "New Article" button → opens editor in create mode
- URL param: ?edit={id} auto-opens editor for that article

ARTICLE EDITOR (components/admin/ArticleEditor.jsx):
- 2-column layout: main content (2/3) + sidebar settings (1/3)
- MAIN CONTENT:
  - Title input (large, Playfair font)
  - Excerpt textarea
  - Rich text editor: ReactQuill with toolbar (headings, bold, italic, underline, blockquote, lists, link, image, clean)
- SIDEBAR:
  - Cover image: upload via file input → Base44 UploadFile integration, preview with remove button
  - Category select (Legal Insights, Opinion, Case Study, News, Explainer)
  - Author text input
  - Publish date picker (type=date)
  - Published toggle (Switch)
  - Featured toggle (Switch)
  - Instagram link input
  - Video URL (embed) input
- Header: Back arrow, article title, Save button (crimson, uppercase)
- Auto-generates slug from title (lowercase, hyphens)

================================================================================
7. ADDITIONAL TECHNICAL NOTES
================================================================================

- Authentication: handled by Base44 platform (no custom login page needed)
- Data fetching: TanStack Query (useQuery, useMutation, queryClient.invalidateQueries)
- All article filtering by is_published=true for public pages
- Admin pages fetch all articles regardless of publish status
- Page title: "Clair De Loi" (set in index.html)
- Meta description: "Illuminating the law, one story at a time. Editorial excellence in legal discourse."
- Favicon: Clair De Loi logo JPEG
- Icons: Lucide React only (no inline SVGs — causes SVGAnimatedString cloning errors)
- Custom scrollbar: 6px, crimson thumb
- Rich text editor global CSS overrides: Quill toolbar and container border-colors match design system
- Reading progress bar: scroll event listener, calculated as scrollY / (scrollHeight - innerHeight) * 100
